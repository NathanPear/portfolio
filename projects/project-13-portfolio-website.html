PROJECT 13 / SOFTWARE / DIGITAL

Static ASCII Portfolio Website

A zero-dependency, JSON-driven engineering portfolio designed with a retro ASCII-art terminal aesthetic.

  • Role: system design and front-end engineering
  • Focus: static build automation and performance optimization
  • Constraint: zero-runtime backend and lightweight client-side footprint

Project Details

Problem and Context

Modern portfolios often require complex web frameworks, hydration overhead, and heavy runtimes, leading to slow loading times, maintenance headaches, and poor longevity.

For a multidisciplinary engineering portfolio, the goal was to design a robust, lightning-fast, and visually distinct presentation that emphasizes technical clarity and structured documentation.

Scope and Constraints
  • No runtime backend (fully static hosting compatible with GitHub Pages)
  • Unified retro terminal ASCII design language across all viewports
  • Automated compiler scripts to synchronize portfolio project lists from a single JSON file
  • Responsive mobile-first layout and system-preference-aware dark mode

Approach

What I Built
  • JSON-to-HTML Builder: A Node.js compile script parsing project metadata to generate and inject templated, layout-safe project cards across multiple category pages.
  • ASCII Rule Engine: Dynamic runtime script generating uniform rule lines using repeating ASCII markers (=, -) with custom terminal-style cap characters (+).
  • Theme Controller: Client-side logic for setting and persisting user theme preferences without page flicker.
  • Site Architecture: Minimal CSS grid layouts combined with lightweight media-free SVG graphics to ensure instant page load speeds.
Cross-Disciplinary Read

This project connects software design patterns to systems engineering principles. By treating metadata as the single source of truth, it implements an engineering pipeline for static assets similar to software compilation steps.

Additionally, it emphasizes accessibility and constraint-driven design: by leveraging text characters rather than image-heavy components, it presents information clearly, proving that visual identity can be established with clean layout logic alone.

Outcome

Result

The final site loads instantly, scoring 100% on performance metrics across desktop and mobile. It is highly maintainable, requiring a single npm build command to update links, tags, and cards globally.

It demonstrates the capability to deploy custom web layouts using raw web standard APIs (HTML, CSS, JS) without reliance on heavy dependencies or external framework lock-ins.

Takeaways
  • Separating content (JSON) from presentation (HTML templates) simplifies portfolio maintenance
  • Responsive design and micro-animations can be achieved using vanilla CSS variables
  • Minimizing third-party runtime dependencies results in unmatched loading speeds and security
