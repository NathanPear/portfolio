PROJECT 15 / ELECTRONIC

Ultrasonic Room Scanner

A stepper-motor-driven ultrasonic scanner that measures the geometry around it by sweeping a distance sensor through a full rotation and exporting the readings for 2D reconstruction.

  • Role: sensor control, motion control, and data output
  • Focus: room mapping through angular distance sampling
  • Constraint: keep the sensor aligned and readings usable

Project Details

Problem and Context

The system was built to measure the shape of nearby interior space by scanning distance readings as the sensor rotated through a full sweep.

That makes it a compact sensing-and-control project: the mechanism has to move reliably, the sensor has to return meaningful data, and the output has to be easy to use later.

Scope and Constraints
  • Use an ultrasonic sensor to read distance
  • Rotate the sensor with a stepper motor
  • Support manual clockwise and counterclockwise alignment
  • Export angle-distance data in a MATLAB-ready format

Approach

What I Built
  • A stepper-driven sweep routine that captures geometry over 360 degrees
  • An idle mode for manual sensor alignment using two buttons
  • Filtering logic that fills obvious invalid or inconsistent readings
  • Serial output formatted for downstream shape reconstruction in MATLAB
Cross-Disciplinary Read

This project lives where embedded systems, sensing, and data visualization meet. The hardware has to move smoothly, the sensor data has to be trustworthy, and the output has to be structured enough for analysis.

That makes it easy to understand from either a controls viewpoint or a software-and-data viewpoint, which is useful in a cross-disciplinary portfolio.

Outcome

Result

The final system could sweep a room, record angle and distance samples, filter the measurements, and output data ready for 2D shape reconstruction.

It also supported manual alignment, which made the scanner easier to position before a run.

Takeaways
  • Motion control and sensing are more useful when their data is immediately usable
  • Simple filtering can make raw sensor data much more trustworthy
  • Clear output formats matter when a project feeds into another toolchain
