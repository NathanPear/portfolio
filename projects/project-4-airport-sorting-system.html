PROJECT 4 / INTEGRATED

Airport Sorting System

A simulated luggage transport and sorting system that combines Python routing, a robotic arm, and a custom bridge mechanism.

  • Role: integrated systems coordination
  • Focus: software, mechanism, and robot handoff
  • Constraint: move luggage across multiple subsystems

Project Details

Problem and Context

The assignment was to sort miniature luggage based on passenger ticket data and move it to the right destination platform.

The project mattered because the hard part was not any one subsystem. It was making the software, robot, and mechanism work as a coordinated whole.

Scope and Constraints
  • Parse passenger data in Python
  • Coordinate a robotic arm for staging and movement
  • Bridge the gap between platforms with a custom mechanism
  • Keep the routing logic and physical motion in sync

Approach

What I Built
  • Python-based data parsing and routing decisions
  • Robotic manipulator path coordination
  • Hinged drawbridge motion for cross-platform transport
  • Torque reduction using pulley principles in the mechanism
Cross-Disciplinary Read

This project is a clean example of systems integration. A software reviewer can focus on the routing logic, a mechanical reviewer can look at the transport bridge, and an integrated-systems reviewer can see how the handoffs were managed.

The useful part is the interface between disciplines: each subsystem had to behave correctly on its own and still make sense when chained together.

Outcome

Result

The finished system coordinated routing, robotic movement, and the bridge mechanism so luggage could be sent to the correct destination path.

It demonstrated that the system-level problem was solved, not just the isolated parts.

Takeaways
  • Integrated projects are mostly about coordination and timing
  • Good interfaces reduce friction between subsystems
  • Cross-disciplinary work is easiest to read when the handoffs are explicit
